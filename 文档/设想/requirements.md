# 1. Requirements

## 1.1 Basic

A and B both have the app installed.

- 1.1.1 A and B are close enough. A writes a post, B receives it; vice versa
- 1.1.2 A and B are far away from each other (further than the bluetooth range in theory like 300m). A writes a post, B receives it after getting close enough; vice versa

## 1.2 Other

- 1.2.1 there's no limit in the length of post

# 2. UI

## 1.3 Minimum

- 2.1.1 no timestamp of the post
- 2.1.2 no id of who wrote the post
- 2.1.3 no notification about new posts received is the default setting
- 2.1.4 no percentage or progress bar shown when sync with other peers
- 2.1.6 no support for deleting post
- 2.1.7 no support for searching
- 2.1.8 no support for filtering
- 2.1.9 if there need to be two distributions, one for free and one paid, two versions have the exact same UI and features
- 2.1.10 no connection status

## 2.2 List of Posts
 
- 2.1.1 all the posts aligned to the right side, as most chat apps put the messages written by self on the right side
- 2.1.2 latest post on top
- 2.1.3 user can copy the content of a selected post
- 2.1.4 every post is shown in full length, no matter how long it is

## 2.2 Text Input

- 2.2.1 only support pure text, no support for pictures, videos, links etc
- 2.2.2 user can paste content into it

## 2.3 "Post" Button

- 2.3.1 when Text Input is empty, it's disabled; otherwise it's enabled
